# PillTracker

휴대폰 브라우저에서 사용할 수 있는 개인용 약 복용 기록 웹 앱입니다.

약 이름, 복용 시간대, 복용 여부, 정확한 복용 시간, 메모를 기록할 수 있고, 오늘의 컨디션도 함께 저장할 수 있습니다. 데이터는 로그인한 사용자별로 서버의 `data/db.json` 파일에 저장됩니다.

가입은 이메일 없이 ID와 비밀번호만 사용합니다. ID는 가입 화면에서 중복 여부를 확인합니다.

자동 로그인은 기본으로 켜져 있고, 체크를 해제하면 브라우저를 닫을 때 사라지는 세션 쿠키로 로그인합니다. 자동 로그인을 켠 경우 로그인 쿠키는 30일 동안 유지됩니다.

## 주요 기능

- ID와 비밀번호만으로 가입 및 로그인
- ID 중복 확인
- 자동 로그인 기본 활성화 및 해제 옵션
- 약 복용 기록 추가, 수정, 삭제
- 졸림 정도, 집중도, 종합컨디션, 수면 시간, 카페인 여부 기록
- 날짜별 이전 기록 조회
- 사용자별 서버 저장
- 선택형 서버 저장 암호화

## 실행 방법

이 프로젝트는 프레임워크 없이 HTML, CSS, JavaScript와 Node.js 내장 모듈만 사용합니다.

1. Node.js 18 이상이 설치되어 있는지 확인합니다.
2. 터미널에서 이 폴더로 이동합니다.
3. `npm start`를 실행합니다.
4. 브라우저에서 `http://localhost:3000`으로 접속합니다.
5. 가입 또는 로그인 후 기록을 입력합니다.

## 테스트

Playwright로 실제 브라우저 UI 테스트를 실행할 수 있습니다.

```bash
npm test
```

브라우저가 열리는 모습을 보면서 실행하려면 다음 명령을 사용합니다.

```bash
npm run test:ui
```

테스트 리포트는 다음 명령으로 확인합니다.

```bash
npm run test:report
```

## 데이터 저장

서버를 처음 실행하면 `data/db.json` 파일이 자동으로 만들어집니다. 이 파일에는 사용자 계정, 비밀번호 해시, 로그인 세션, 복용 기록과 컨디션 기록이 저장됩니다.

비밀번호는 원문으로 저장하지 않고 PBKDF2 해시로 저장합니다.

`data/db.json`은 `.gitignore`에 포함되어 있어 GitHub에 업로드되지 않습니다.

## 보안 기능

- 비밀번호 원문 미저장 및 사용자별 salt 기반 PBKDF2 해시 저장
- `HttpOnly`, `SameSite=Lax` 세션 쿠키 사용
- HTTPS 환경에서 쿠키 `Secure` 속성 자동 적용
- 로그인, 가입, ID 중복 확인 요청 rate limit
- 데이터 저장 및 로그아웃 요청 CSRF 토큰 검사
- 상태 변경 요청의 `Origin` / `Referer` 검사
- `Content-Security-Policy`, `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy` 보안 헤더 적용
- 선택형 클라이언트 측 암호화: 보안 설정에서 켜면 기록은 로그인 비밀번호로 브라우저에서 AES-GCM 암호화된 뒤 서버에 저장됩니다.

로그인 비밀번호는 원문으로 저장되지 않으며, 서버 저장 암호화의 복호화 키도 서버에 저장되지 않습니다. 로그인 비밀번호를 잃어버리면 서버 운영자도 암호화된 기록을 복구할 수 없습니다.

## 휴대폰에서 테스트하는 방법

컴퓨터와 휴대폰이 같은 Wi-Fi에 연결되어 있어야 합니다.

1. `npm start`로 앱을 실행합니다.
2. 컴퓨터의 로컬 IP 주소를 확인합니다.
3. 휴대폰 브라우저에서 `http://컴퓨터IP:3000` 형태로 접속합니다.
4. 예: `http://192.168.0.10:3000`
